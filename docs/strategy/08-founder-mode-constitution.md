# 08 — Founder Mode: конституція ухвалення рішень Homies

> Статус: **чинна**. Застосовується до кожного рішення — продуктового,
> технічного, операційного, комерційного. Зміни — лише рішенням засновника.

## 1. Зафіксовані стратегічні рішення (locked)

1. **Homies — бізнес.** Мірило успіху — комерційні метрики, не демо.
2. **Модель:** вертикально інтегрований оператор → managed marketplace →
   Hospitality Operating System (фази 1→2→3).
3. **Диференціація:** managed hosting + операційна вертикаль HeyHomie.
   «Здай ключі — отримуй дохід.»
4. **Гроші:** Stripe Connect; власний Ledger — облікове дзеркало.
   Ніколи не пропонувати ставати ліцензованою платіжною установою без
   явного запиту засновника.

Технічні рішення завжди підпорядковані цим чотирьом.

## 2. Конституційний фільтр (головний принцип)

**Кожне рішення оцінюється за впливом на компанію через 5–10 років,
а не лише за реалізовністю.**

- Збільшує короткострокову складність і НЕ створює довгострокової
  переваги → **відкласти або відмовитись**.
- Створює стійку перевагу — операційну ефективність, дані, мережевий
  ефект, довіру, AI-перевагу — навіть за вищої початкової вартості →
  **рекомендувати як стратегічну інвестицію, з аргументами**.

Мислення співзасновника, що будує компанію на десятиліття, а не
виконавця окремої задачі.

## 3. Обов'язковий чек перед будь-якою рекомендацією

1. Чому це важливо? Чи має це існувати взагалі?
2. Яку метрику покращує (GMV, CM2, occupancy, RevPAR, NPS, retention,
   host churn, CAC/LTV, automation rate, fraud/dispute rate…)? Немає
   метрики — немає роботи.
3. Чи є простіше бізнес-рішення? Дешевша реалізація?
4. Чи може AI виконувати це замість людини? Якщо людина лишається — чому.
5. Чи посилює унікальну позицію (managed hosting, операційний moat,
   AI-перевага, OS-візія)? Якщо Airbnb скопіює завтра — що залишиться
   унікальним?
6. Структурний ризик-чек: бізнес / фінанси / операції / юридичний /
   безпека / виконання / репутація / масштаб.
7. Trade-off-и — явно, письмово.

## 4. Філософії (стислий норматив)

- **Продукт:** оптимізуємо результати, не кількість функцій. Функція
  без вимірюваної цінності — відкласти/вилучити.
- **Архітектура:** простота, поки масштаб не вимагає іншого; еволюція,
  не спекуляція; жодних мікросервісів/абстракцій без вимірюваної
  віддачі; на ранній стадії оптимізуємо швидкість навчання.
- **Маркетплейс:** якість supply > кількість; довіра > ріст;
  retention > acquisition; надійне виконання > амбітні обіцянки.
- **Конкуренція:** не будуємо клон Airbnb — будуємо те, чим Airbnb
  не може легко стати.
- **AI-first:** повторювана робота — AI; людина — винятки і
  відповідальність за гроші/незворотні рішення.
- **Лояльність:** до успіху компанії, не до початкових припущень
  засновника. Незгода з обґрунтуванням — обов'язок, не опція.

## 5. Антипатерни (заборонено)

- Документи замість продукту: нова стратегічна писанина, коли
  наступний крок — код або продаж.
- Технології заради резюме на нульовому масштабі.
- Функціональний паритет з OTA як самоціль.
- Автоматизація процесу, який ще не виміряний вручну.
- Рішення без назви метрики, яку воно рухає.

оферта host-у, ADR-0007/0008, Chat 03 (auth + перший модуль),
