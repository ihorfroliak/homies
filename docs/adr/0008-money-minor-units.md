# ADR-0008: Гроші — цілим числом у найменших одиницях

- **Статус:** Accepted
- **Дата:** 2026-06-25

## Контекст
Дробові числа (float) у комп'ютері зберігаються неточно: класичний приклад `0.1 + 0.2` дає `0.30000000000000004`. Для звичайних обчислень це дрібниця, але для **грошей** такі похибки накопичуються й неприпустимі в реальному бізнесі з податковою звітністю.

## Рішення
Усі грошові суми зберігаємо й передаємо як **ціле число в найменших одиницях валюти** (грошах): `350000` = `3500.00 PLN`. Поруч завжди йде код валюти (ISO 4217). Це та сама конвенція, що у Stripe. Жодних `float`/`double` для грошей — ні в REST-контрактах, ні в подіях.

## Наслідки
**Позитивні:** точність без накопичення похибок; узгодженість між REST-API та подіями; сумісність зі Stripe-конвенцією.

**Негативні:** на межі відображення треба ділити на 100 для форматування в UI; розробники мають пам'ятати конвенцію (тому вона задокументована прямо в контрактах і схемах).

## Розглянуті альтернативи
**float/double** — просто, але неточно; відкинуто. **Decimal у вигляді рядка** — теж точно, але незручніше в арифметиці й різні бекенди по-різному його парсять.
