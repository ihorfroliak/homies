# ADR-0003: Python + FastAPI як основна мова бекенду

- **Статус:** Accepted
- **Дата:** 2026-06-25

## Контекст
Потрібна мова бекенду, яка дає швидку розробку, надійну типізацію контрактів і **синергію з аналітикою та ML** — бо проєкт навмисно демонструє Data Engineering і Data Science поряд із бекендом.

## Рішення
Основна мова — **Python + FastAPI**: типізація через Pydantic, автоматична генерація OpenAPI зі коду, велика екосистема для роботи з даними й ML. Опційно один сервіс можна написати на **Go** — щоб показати поліглотність і високопродуктивний компонент.

## Наслідки
**Позитивні:** одна мова через бекенд, дані й ML — менше контекст-перемикань; дуже швидка розробка фіч; першокласний OpenAPI-тулінг «з коробки».

**Негативні:** не найвища «сира» обчислювальна продуктивність (CPU-важкі задачі варто виносити в окремі воркери); GIL обмежує паралелізм у межах процесу.

## Розглянуті альтернативи
**Go** — швидший і легший у деплої, але далі від екосистеми Data Science і повільніша розробка продуктових фіч. **Node.js/TypeScript** — добра швидкість і одна мова з фронтендом, але слабша синергія з DS/ML, яка є метою проєкту.
