## 5. Пріоритезований план покращень (Enterprise Improvement Roadmap)

**Крок 0 — рішення (тиждень, без коду):**
1. Критерій успіху: портфоліо чи бізнес → визначає все нижче.
2. ADR-0007: гроші через регульованого партнера (Stripe Connect
   рекомендовано: Connect+destination charges покривають escrow-механіку,
   спліти, KYC host-ів частково, звітність) — Ledger стає обліковим
   дзеркалом.
3. ADR-0008: доступність як інтервали + exclusion constraint; календар
   днів — read-проєкція.
4. Зафіксувати диференціацію: managed hosting + HeyHomie-вертикаль
   (документ 1 сторінка).

**Крок 1 — технічний фундамент (разом із Chat 03):**
outbox-патерн, ідемпотентні консюмери, Alembic, архітектурні тести
ізоляції, rate limiting, security headers, SCA в CI, .env.example,
синхронізація контрактів (T1, T6).

**Крок 2 — до першої реальної транзакції:**
MFA адмінів, IBAN cooling period, затримка виплат новим host-ам,
верифікація об'єкта, PITR-бекапи+тест відновлення, базові SLO+алерти,
runbook-и двох головних відмов, VAT-консультація.

**Крок 3 — до публічного запуску:**
сідінг supply + гарантії host-ам, iCal-sync, Apple/Google Pay + guest
checkout, SEO-каркас, B2B-пілот, AI-хвиля 1, метрик-словник,
регуляторний реєстр.

**Крок 4 — масштабування:** PM-пакет, graph-антифрод, ES для Ledger,
experimentation, referral, друга країна лише після liquidity-цілі в
першому місті.

## 6. Підсумок ради

Архітектурне мислення випереджає стадію проєкту — це і сила (портфоліо,
менше переробок), і головна загроза (аналіз замість продукту). Три
рішення P0 (гроші/ліцензія, диференціація, критерій успіху) коштують
нуль рядків коду і визначають, чи має сенс решта. Рада наполягає:
жодного нового пакета документації, поки не працює перший end-to-end
booking happy path з грошима в sandbox.
