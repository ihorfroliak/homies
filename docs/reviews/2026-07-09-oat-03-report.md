## 3. Regression — money core intact

Warfare on live Postgres (worker running): all D6 verdicts PASS

## 4. Bugs discovered + root causes

None new this cycle. The double-capture race (found in OAT-02) stays fixed
(single-capture PASS). Delivery failures are fully isolated from money — a
dead notification leaves booking/payment/ledger untouched (proven).

## 5. Risk register delta

**Closed by OAT-03:**
- Durable delivery (was log-only, lost on crash) → outbox, survives restart.
- No retry model → exponential backoff + jitter + max attempts + DEAD.
- No delivery observability → Prometheus metrics + founder audit endpoints.
- Duplicate-worker risk → SKIP LOCKED, proven disjoint.
- Silent loss → nothing disappears: delivered OR observable as DEAD.

**Still open:**
| Gap | Status | Priority |
|---|---|---|
| Real email/SMS (channels are stub/SMTP-untested) | SMTP adapter exists, no live provider creds | P1 |
| Exactly-once across external provider | at-least-once + idem key (honest) | acceptable pilot |
| Check-in instruction content (codes/keys) | event/template exist; payload minimal | P1 |
| Full Support (S6) / Disputes (S8) | still absent | P1/P2 |
| Auto-void unpaid (ghost-booking) | still open | P1 |
| Worker HA (single process) | fine for pilot; multi-instance later | P3 |

## 6. Release Gate update

Notification delivery: best-effort → **durable, retryable, observable**.
The pilot's guest/host/founder now get notifications that survive crashes and
are auditable; a failed provider degrades gracefully without touching money.
Readiness ~56 → ~62. Next-highest ROI: real email provider creds (swap
`EMAIL_PROVIDER=smtp`) + check-in instruction content.

## 7. Success condition — met

> Booking committed → Event committed → Outbox committed, then regardless of
> crashes/restarts/retries/provider failures, the notification is eventually
> delivered OR permanently observable as DEAD. Nothing disappears silently. No
> business transaction depends on external providers.

All four clauses proven by executable evidence above.
